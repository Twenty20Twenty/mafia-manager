## Тесты алгоритма рассадки

### S-01: Базовая рассадка, 30 участников, 1 тур
- 30 игроков → 3 стола × 10 слотов = 30 слотов
- Каждый игрок встречается ровно 1 раз

### S-02: Уникальность слотов — 10 туров, 10 игроков
- При 10 турах и 10 игроках каждый игрок занимает все слоты (1-10) ровно по 1 разу
- Проверяет алгоритм минимизации повторных слотов

### S-03: Более 11 туров, 30 участников — фоллбэк
- После 10 уникальных слотов алгоритм использует фоллбэк (любой свободный слот)
- 12 туров генерируются без ошибок
- В каждом туре по-прежнему все 30 игроков уникальны

### S-04: Исключение игрок–игрок (2 исключения)
- Два игрока с исключением никогда не сидят за одним столом
- Проверяется на 3 турах

### S-05: Исключение игрок–судья стола (2 исключения)
- Игрок с исключением не попадает за стол данного судьи
- Проверяется по table_number в сгенерированных играх

### S-06: Смешанная рассадка, 50 участников
**Сценарий:**
1. Генерируем туры 1-7 (обычные балансированные)
2. Туры 8-11 генерируются по швейцарке (тиры 30+20)
3. 2 исключения игрок-игрок + 1 исключение игрок-судья

**Проверки:**
- После 7 обычных туров: 7×5=35 игр
- После 4 швейцарских: итого 11×5=55 игр
- Исключения соблюдены во всех 55 играх
- В каждой игре ровно 10 слотов

### S-07: Финальная рассадка
- 10 зафиксированных финалистов → 1 игра с stage=final_round
- Без зафиксированных → RuntimeException
- Менее 10 финалистов → RuntimeException

### S-08: Исправление бага (roundsCount)
- Без швейцарки и при roundsCount=5 генерируется ровно 5 туров
- Номера туров не превышают roundsCount

### S-09: Командный турнир
- Игроки одной команды не сидят за одним столом
- 2 команды по 10 игроков → 5+5 на стол

## Запуск тестов

```bash
# Все тесты
mvn test

# Только тесты рассадки
mvn test -Dtest=SeedingServiceTest

# Только тесты авторизации
mvn test -Dtest=AuthServiceTest

# Все тесты сервисного слоя
mvn test -Dtest="*ServiceTest"
```
